# Отчет

## Rolling Hash

Rolling hash — это техника вычисления хеша для скользящего окна, позволяющая пересчитывать хеш за O(1) при сдвиге окна.

**Суть работы:**

- Для строки `s` длины `m` хеш вычисляется как полиномиальная функция:  
  `hash(s) = (s[0]·p^(m-1) + s[1]·p^(m-2) + ... + s[m-1]·p^0) mod q`
- При сдвиге окна на один символ:  
  `new_hash = (old_hash - s[i]·p^(m-1))·p + s[i+m]`
- Предвычисление `p^(m-1) mod q` позволяет эффективно удалять вклад уходящего символа

**Преимущество:** избегаем O(m) операций на каждое окно, получая O(1) на пересчет хеша.

## Сложность алгоритма

**Временная сложность:**

- **Средний случай:** O(n + m)  
  Вычисление хешей за O(n), проверка совпадений при коллизиях в среднем O(1)
- **Худший случай:** O(n·m)  
  Возникает при частых коллизиях хешей (специально подобранные данные)

**Сложность по памяти:** O(1)  
Дополнительная память не зависит от размера входных данных (только переменные для хешей и предвычисленных значений).
